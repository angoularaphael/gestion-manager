NE COLLE PAS index.html entier dans WordPress — ça donne une page noire.

ÉTAPE 1 — Envoyer les fichiers sur OVH
---------------------------------------
1. OVH → Hébergements → Multisite → Gestionnaire de fichiers
2. Ouvre : www/wp-content/uploads/
3. Crée le dossier : offre-d-ete
4. Envoie dedans :
   - index.html
   - le dossier assets/ (complet)

ÉTAPE 2 — Vérifier
------------------
Ouvre dans le navigateur :
https://boxingcenter.fr/wp-content/uploads/offre-d-ete/index.html

Tu dois voir la promo (gants puis 89€). Si 404 → fichiers mal placés.

ÉTAPE 3 — Dans WordPress (page Offre d'été)
--------------------------------------------
Colle UNIQUEMENT le contenu de wordpress-iframe.html

→ Charge index.html depuis Vercel (même version que public/offre-d-ete/)
→ Scripts partagés : assets/embed-config.js + assets/embed-parent.js
→ Hauteur auto (hero, FAQ, t-shirt, CTA) via postMessage

Enregistrer → tester : https://boxingcenter.fr/offre-d-ete

POURQUOI LA PAGE NOIRE ?
-------------------------
1. WordPress bloque les <script src="..."> externes → l'iframe restait vide.
   → wordpress-iframe.html met maintenant le src DIRECT sur l'iframe + script inline.

2. L'intro noire (animation gants) peut bloquer l'affichage.
   → L'URL ?embed=1 saute l'intro sur WordPress (contenu visible tout de suite).

3. Si toujours noir : vider le cache WordPress / navigateur, puis tester :
   https://gestion-manager.vercel.app/offre-d-ete/index.html?embed=1
